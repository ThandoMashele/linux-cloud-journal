## 🧠 Key Things I Learned
- `pwd` = print working directory — always know where you are
- `~` = home folder, `..` = up one level, `.` = current folder
- `>` overwrites a file, `>>` appends — never mix these up
- `rm` has no recycle bin — deleted files are gone permanently
- Linux is case-sensitive — `Documents` and `documents` are different
- Spaces in filenames require quotes — better to just use hyphens
- `mkdir -p` creates nested folders in one command
- Always check `pwd` before running cp, mv or rm
- You can't copy a folder into itself — go up one level first
- `grep -B/-A` shows context lines around matches
- Bash variables: no spaces around `=` ever (`name="value"` not `name = "value"`)
- `if` needs `fi`, `for` needs `done` — every block must be closed
- `[ "$var" = "value" ]` needs spaces inside the brackets
- `#!/bin/bash` shebang must be the first line of every script
- Use `cat > file << 'EOF'` for writing multi-line scripts
- `$(command)` captures command output into a variable
